


\chapter{Proposed System}
This chapter discusses the details of the proposed CyberSentinel system, which provides advanced URL threat detection through machine learning algorithms, heuristic analysis, and community-driven feedback mechanisms.

\section{Analysis/Framework/Algorithm}

\subsection{System Framework}
CyberSentinel employs a hybrid detection framework that integrates multiple analysis layers to achieve comprehensive threat identification. The system architecture combines:

\begin{enumerate}
	\item \textbf{Heuristic Analysis Layer}: Provides immediate baseline protection through rule-based pattern matching and statistical analysis of URL characteristics.
	
	\item \textbf{Machine Learning Classification Layer}: Implements supervised learning algorithms trained on extensive datasets containing 651,191 URLs across four threat categories.
	
	\item \textbf{Community Intelligence Layer}: Incorporates user feedback and threat reporting mechanisms for continuous model improvement and validation.
	
	\item \textbf{Real-time Processing Layer}: Ensures sub-100ms response times through optimized feature extraction and model inference pipelines.
\end{enumerate}

\section{Design Details}

\subsection{System Architecture}
CyberSentinel follows a microservices architecture pattern with clear separation of concerns:

\textbf{Frontend Layer (React + Tailwind CSS):}
\begin{itemize}
	\item Responsive single-page application with routing
	\item Real-time URL analysis interface with instant feedback
	\item Community feedback forms for threat reporting
	\item Analytics dashboard for threat statistics visualization
\end{itemize}

\textbf{API Gateway Layer (Node.js + Express):}
\begin{itemize}
	\item RESTful API endpoints for URL analysis and feedback collection
	\item Request validation, rate limiting, and security middleware
	\item Session management and user authentication
	\item Database interaction and data persistence
\end{itemize}

\textbf{Machine Learning Service (Flask + Python):}
\begin{itemize}
	\item Model serving with optimized inference pipeline
	\item Feature extraction and preprocessing modules
	\item Model training and continuous learning capabilities
	\item Performance monitoring and model versioning
\end{itemize}

\textbf{Database Layer (MongoDB):}
\begin{itemize}
	\item URL analysis logs with indexing for performance
	\item User feedback storage with validation mechanisms
	\item Model training data and feature engineering results
	\item System analytics and performance metrics
\end{itemize}

\subsection{User Interface Design}
The system interface prioritizes usability and accessibility:

\textbf{Main Analysis Interface:}
\begin{itemize}
	\item Clean, professional URL input form with validation
	\item Real-time analysis results with confidence scoring
	\item Color-coded threat classification (Benign, Defacement, Malware, Phishing)
	\item Detailed analysis breakdown with educational tooltips
\end{itemize}

\textbf{Feedback Interface:}
\begin{itemize}
	\item Intuitive threat reporting forms with guided workflows
	\item Community impact statistics and contribution tracking
	\item Educational resources for cybersecurity awareness
	\item User-friendly error handling and confirmation messages
\end{itemize}

\section{Methodology}
This chapter outlines the systematic approach employed in developing the CyberSentinel URL threat detection system, including the machine learning methodology, system architecture, and implementation strategy.



\subsection{Model Architecture}
The CyberSentinel system employs a fine-tuning approach based on pretrained Large Language Models (LLMs) specifically adapted for URL threat detection. Figure \ref{fig:model_architecture} illustrates the complete fine-tuning process.\cite{llm}.

\begin{figure}[H]
	\centering
	\includegraphics[width=0.9\textwidth]{images/URL-Detection.png}
	\caption{Fine-Tuning Architecture for URL Threat Detection Using Pretrained Language Models}
	\label{fig:model_architecture}
\end{figure}

The architecture leverages the contextual understanding capabilities of pretrained language models, which have learned comprehensive language structures and patterns from billions of words. Through fine-tuning, the model adapts this knowledge to understand URL characteristics and predict threat classifications across four categories: benign, defacement, malware, and phishing.
